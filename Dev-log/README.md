# 개발일지

알고리즘 저지 사이트 프로젝트의 개발일지를 기록하는 공간입니다.

## 파일명 규칙
- 날짜 기반: `YYYY-MM-DD.md` 형식
- 예시: `2024-01-15.md`

## 작성 형식
각 개발일지 파일은 다음 형식을 따릅니다:

1. **날짜**: 작업한 날짜
2. **작업 내용**: 백엔드/프론트엔드로 구분하여 작성 (각 작업에 커밋 링크 포함)
3. **구현한 기능**: 새로 추가하거나 수정한 기능 (백엔드/프론트엔드 각각)
4. **해결한 이슈**: 버그 수정이나 문제 해결 내용 (백엔드/프론트엔드 각각)
5. **관련 커밋**: 해당 날짜의 모든 커밋 목록

## 개발일지 작성 방법

### AI 어시스턴트를 통한 작성 (현재 사용 중)

AI 어시스턴트에게 특정 날짜의 커밋을 확인하여 개발일지를 작성하도록 요청합니다.

#### 사용법

1. AI 어시스턴트에게 다음과 같이 요청합니다:
   ```
   오늘 날짜(또는 특정 날짜)에 해당하는 커밋을 보고 개발일지를 작성해주세요.
   ```

2. AI 어시스턴트가 자동으로:
   - 해당 날짜의 Git 커밋을 확인
   - 커밋의 변경 내용을 분석하여 작업 내용 요약
   - 백엔드/프론트엔드로 자동 분류
   - 각 작업에 해당하는 커밋 링크 추가
   - 구현한 기능과 해결한 이슈 추출
   - 템플릿 형식에 맞춰 개발일지 작성

#### 커밋 연결

- 각 작업 내용 옆에 해당 커밋의 해시와 링크가 자동으로 추가됩니다.
- 파일 끝에 "관련 커밋" 섹션에 해당 날짜의 모든 커밋 목록이 포함됩니다.
- 커밋 링크를 클릭하면 GitHub에서 해당 커밋의 상세 내용을 확인할 수 있습니다.

#### 예시

```markdown
### 백엔드
- 사용자 인증 API 구현 [`a1b2c3d`](https://github.com/.../commit/a1b2c3d)
- 데이터베이스 연결 설정 추가 [`e4f5g6h`](https://github.com/.../commit/e4f5g6h)
```
